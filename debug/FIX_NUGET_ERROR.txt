NuGet Connectivity Error Fix Instructions
==========================================

ERROR MESSAGE:
"Unable to load the service index for source https://api.nuget.org/v3/index.json"

This is NOT a code error - the indicator code is correct.
cTrader cannot connect to NuGet package repository.

SOLUTIONS:
----------

1. CHECK INTERNET CONNECTION
   - Verify you have active internet connection
   - Try opening https://api.nuget.org in a browser
   - Check if firewall is blocking api.nuget.org

2. RETRY BUILD IN cTrader
   - Close and reopen cTrader Automate
   - Click "Build" again
   - Sometimes temporary network glitches resolve themselves

3. CLEAR NUGET CACHE (Windows)
   - Open Command Prompt as Administrator
   - Run: dotnet nuget locals all --clear
   - Restart cTrader and try building again

4. CHECK PROXY/VPN SETTINGS
   - If you're behind a corporate firewall or using VPN
   - Configure NuGet to use your proxy settings
   - Or temporarily disable VPN and try again

5. OFFLINE BUILD (Advanced)
   - The indicator only uses built-in cTrader APIs
   - No external packages should be needed
   - If issue persists, this might be a cTrader environment issue

WHAT THE INDICATOR USES:
------------------------
- System (built-in .NET)
- System.Collections.Generic (built-in .NET)
- System.Linq (built-in .NET)
- cAlgo.API (cTrader built-in)
- cAlgo.API.Indicators (cTrader built-in)

NO EXTERNAL PACKAGES REQUIRED!

If none of these work, the issue is with cTrader's build environment configuration,
not with the indicator code itself.
