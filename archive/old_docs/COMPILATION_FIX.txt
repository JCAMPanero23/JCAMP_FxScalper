Issue: Variable redefinition errors
Date: 2026-03-03

───────────────────────────────────────────────────────────────────────
Problem
───────────────────────────────────────────────────────────────────────

Include files (.mqh) were declaring variables as "extern" which
conflicted with "input" declarations in the main EA file.

Error example:
"variable 'RiskPercent' already defined with different type"

───────────────────────────────────────────────────────────────────────
Solution Applied
───────────────────────────────────────────────────────────────────────

Removed all "extern" variable declarations from include files:

✓ JC_Utils.mqh
  Removed: TradeLondon, TradeNewYork, TradeAsian, TradeTokyo, MaxSpread

✓ JC_RiskManager.mqh
  Removed: RiskPercent, MaxDailyLoss, MaxGlobalPositions

✓ JC_MarketStructure.mqh
  Removed: LevelProximity, EnableTPValidation, EnableSLSnapping, ATR_Period

✓ JC_EntryLogic.mqh
  Removed: SMA1_Period, SMA2_Period, SMA3_Period, RSI_Period

───────────────────────────────────────────────────────────────────────
Technical Explanation
───────────────────────────────────────────────────────────────────────

In MQL5:
- Input parameters should ONLY be declared once in the main EA file
- Include files can use these variables without declaring them
- The "input" declaration in the main file makes variables globally
  accessible to all included files

Before (Incorrect):
  Main EA: input double RiskPercent = 1.0;
  Include:  extern double RiskPercent;  ← CONFLICT!

After (Correct):
  Main EA: input double RiskPercent = 1.0;
  Include:  (no declaration needed - just use RiskPercent directly)
